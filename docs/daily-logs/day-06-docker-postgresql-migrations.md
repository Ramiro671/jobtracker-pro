## docker-compose.yml

```yaml
services:
  db:
    image: postgres:16-alpine
    container_name: jobtracker_db
    environment:
      POSTGRES_DB: jobtracker_dev
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
    volumes:
      - jobtracker_pgdata:/var/lib/postgresql/data

volumes:
  jobtracker_pgdata:
```

**Commands:**

## EF Core Migrations

```bash
# Install EF Core CLI tool (global)
dotnet tool install --global dotnet-ef

# Add Design package to both projects
dotnet add src/JobTrackerPro.Infrastructure package Microsoft.EntityFrameworkCore.Design
dotnet add src/JobTrackerPro.Api package Microsoft.EntityFrameworkCore.Design

# Create migration
dotnet ef migrations add InitialCreate \
  --project src/JobTrackerPro.Infrastructure \
  --startup-project src/JobTrackerPro.Api

# Apply migration to database
dotnet ef database update \
  --project src/JobTrackerPro.Infrastructure \
  --startup-project src/JobTrackerPro.Api
```

**Result:** `Done.` — Tables created in PostgreSQL:
- `"JobApplications"` — with TechStack columns (owned entity)
- `"Companies"` — with unique index on Name
- `"Users"` — with unique index on Email
- `"__EFMigrationsHistory"` — EF Core internal tracking table

### Bug 2 — TechStack has no primary key
```
Error: The entity type 'TechStack' requires a primary key to be defined.
Fix: Configure TechStack as an Owned Entity using OwnsOne() in EF Core configuration
```

**Key concept:** Value Objects don't have their own table.
`OwnsOne()` tells EF Core to store `TechStack` columns inside the `JobApplications` table:

```csharp
builder.OwnsOne(j => j.TechStack, ts =>
{
    ts.Property(t => t.Backend).HasColumnName("TechStack_Backend");
    ts.Property(t => t.Frontend).HasColumnName("TechStack_Frontend");
    // ...
});
```

## What is Docker? (concept learned today)

Docker packages an application and all its dependencies into a **container** — an isolated, reproducible environment.

```
Without Docker:
  Install PostgreSQL on Windows → configure user → create DB → manage service
  (different on every machine, easy to break)

With Docker:
  docker-compose up -d ← one command, same result everywhere
```

**Key terms:**
| Term | What it is |
|------|-----------|
| `image` | Blueprint for a container (postgres:16-alpine) |
| `container` | Running instance of an image (jobtracker_db) |
| `volume` | Persistent storage — data survives container restart |
| `port mapping` | `5432:5432` → local port 5432 talks to container port 5432 |
| `-d` flag | Detached mode — runs in background |
