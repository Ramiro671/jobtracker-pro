## Key Formulas

### 1. Basic Passive Voice
> **Subject + To Be (conjugated) + Past Participle**

- *The API was built using ASP.NET Core.*
- *The database was migrated using EF Core.*
- *The token was generated by the auth service.*

### 2. Passive Voice by Tense
> **To Be (tense) + Past Participle**

| Tense | Formula | Example |
|---|---|---|
| Simple Present | `is/are + PP` | *The logs are stored in Redis.* |
| Simple Past | `was/were + PP` | *The schema was migrated last night.* |
| Present Perfect | `has/have been + PP` | *The bug has been fixed.* |
| Future | `will be + PP` | *The API will be deployed tomorrow.* |
| Modal | `modal + be + PP` | *The token must be validated.* |

### 3. Impersonal Passive
> **It + be + Past Participle (reporting verb) + that + clause**

- *It is believed that JWT is the standard for authentication.*
- *It is expected that the tests will pass.*
- *It was reported that the pipeline failed.*

### 4. Personal Passive
> **Subject + Passive Verb + to-Infinitive**

- *JWT is believed to be the standard for authentication.*
- *Clean Architecture is known to improve maintainability.*
- *Redis is considered to be essential for caching.*

### 5. Impersonal → Personal Transformation
```
It is said that EF Core has a learning curve.
         ↓
EF Core is said to have a learning curve.
```

---

## When to Use the Passive Voice

- The agent (doer) is **unknown** → *The window was broken.*
- The agent is **unimportant** → *The results were published.*
- To **emphasize the result** → *The API was deployed successfully.*
- In **formal / technical writing** → *The sample was tested at runtime.*
- To **avoid assigning blame** → *Mistakes were made.*
